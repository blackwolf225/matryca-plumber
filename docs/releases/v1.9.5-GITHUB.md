## Two-tier agent discipline

Matryca Plumber now documents an explicit **LLM OS** contract for external agents:

| Tier | Role |
|------|------|
| **Tier 1 — Gardener** | Daemon Phase 1 harvest (`matryca plumber start`) — builds `[[Matryca Master Index]]` |
| **Tier 2 — Cognitive Agent** | MCP / CLI agents — must not impersonate the Gardener |

Full cognitive law: [`SYSTEM_PROMPT.md`](../../SYSTEM_PROMPT.md) § "LLM OS". Distribution pointer: [`llms.txt`](../../llms.txt) §6.

## Master Index Soft Gate (Human-in-the-Loop)

When Phase 1 is incomplete, Tier-2 agents **pause** and present three cost-transparent options:

1. **Local Daemon (recommended)** — run `uvx matryca-plumber plumber start` with a local LLM.
2. **Blind Search** — proceed with `search_graph` / `bm25` without the index (less precise).
3. **Cloud Indexing** — agent scans the vault now (high token cost).

Agents wait for **explicit authorization** before options 2 or 3.

## `bootstrap_status` — native Phase 1 semaphore

New read target for MCP and CLI — no raw `.matryca_daemon_state.json` scraping:

```bash
uvx matryca-plumber --json read bootstrap_status
```

Returns `bootstrap_complete`, `soft_gate_active`, harvest progress, and catalog health.

## Operator overlay

Provisioning `matryca-l1/` now seeds `llm-os-rules.md` alongside `session-rules.md`.
