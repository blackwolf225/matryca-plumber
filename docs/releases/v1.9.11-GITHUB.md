# GitHub Release — v1.9.11 (copy-paste)

**Title:** v1.9.11 - Sovereign UI Reliability (Large Vaults)

**Tag:** `v1.9.11`

---

## Summary

**Operator UX release:** fixes Sovereign UI timeouts and confusing errors on **large Logseq vaults**. Settings save, graph-path onboarding, L1 provision, and **Start Engine** no longer trigger a full in-process AST bootstrap before responding — the same lazy pattern as v1.9.10 UI startup.

Pre-flight **`warn`** checks (degraded file locks, model not listed) are advisory; only **`fail`** blocks **Start Engine**. The checklist modal auto-dismisses when checks pass.

Operators should upgrade with `uvx matryca-plumber` ≥ **1.9.11**. Point LLM hosts at **`llms.txt` v1.9.11** (§2.0 Plumber commands).

## Highlights

| Area | Change |
|------|--------|
| Settings save | Lazy bootstrap; block save until `GET /api/config` loads; confirm discard on close |
| Start Engine | Lazy bootstrap in UI API; daemon child still eager |
| Pre-flight | `warn` ≠ blocked; modal auto-dismiss when green |
| Graph analytics | 18s server cache, 60s client timeout, offline status on poll failure |
| Config API | `LOGSEQ_GRAPH_PATH` read from `.env` file values |

uvx matryca-plumber --help   # resolves latest wheel from PyPI
