**Title:** v1.9.10 - Sovereign UI Fast Startup

## Summary

**Operator UX release:** the Sovereign UI (`matryca plumber status` / `ui`) binds **`http://127.0.0.1:8500`** in seconds on large vaults by deferring eager AST bootstrap (same lazy pattern as MCP stdio). Graph analytics warm on the first dashboard request.

Documentation now clearly separates **`status`/`ui`** (control room only) from **`plumber start`** (maintenance daemon only) — a common source of “the frontend won't start” confusion.

Operators should upgrade with `uvx matryca-plumber` ≥ **1.9.10**. Point LLM hosts at **`llms.txt` v1.9.10** (§2.0 Plumber commands).

## Highlights

| Area | Change |
|------|--------|
| Sovereign UI | Lazy AST in UI lifespan; skip eager bootstrap in `cli.main` for `status`/`ui` |
| Startup time | Port `8500` + React shell in seconds; analytics on first `/api/graph-analytics` |
| Operator docs | README, CONTRIBUTING, ARCHITECTURE, `llms.txt` §2.0 — **`status` vs `start`** |
| Remaining v1.9.9 docs | Security & Sandbox harmonization (`security-sandbox.md`, OpenSpec index) |

## Upgrade

```bash
uvx matryca-plumber --help   # resolves latest wheel from PyPI
```

**Open the control room (not the daemon):**

```bash
uvx matryca-plumber status
```

**Headless daemon only (no browser):**

```bash
uvx matryca-plumber plumber start
```
