# GitHub Release — v1.10.5 (copy-paste)

**Title:** v1.10.5 - Logseq Matryca Parser 1.3.1 Alignment

**Tag:** `v1.10.5`

---

## Summary

**Parser alignment patch:** raises the minimum **`logseq-matryca-parser`** dependency to **1.3.1** and aligns Plumber imports with the consolidated public API introduced in parser **1.3.0**.

**Why upgrade:** inherit graph parity from parser **1.2.x** (YAML frontmatter page properties, case-insensitive page routing, multimodal asset extraction, round-trip serialization fixes) and the stable root exports (`LogosParser`, `SessionAliasRegistry`, `discover_graph_files`). AST cache bootstrap telemetry now uses the parser's file discovery (same exclusions as `LogseqGraph.load_directory`).

Operators should upgrade with `uvx matryca-plumber` ≥ **1.10.5**. Point LLM hosts at **`llms.txt` v1.10.5**.

## Highlights

| Area | Change |
|------|--------|
| Parser dep | `logseq-matryca-parser>=1.3.1,<2.0.0` (was `>=1.1.1`) |
| Spatial reads | Root-level `LogosParser` import in `matryca_hooks` |
| AST cache | Bootstrap file count via `discover_graph_files` |
| Headless writes | Splice index uses parser `_insertion_line_after_node` |

## Changed

- **`logseq-matryca-parser`** — minimum **1.3.1**; graph parity 1.2.x + public API 1.3.0.
- **Parser import alignment** — `matryca_hooks`, `ingestion`, `conftest` use root exports; `ast_cache` uses `discover_graph_files`.
