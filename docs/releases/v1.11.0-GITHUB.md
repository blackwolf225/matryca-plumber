## Summary

**Headline:** import your **Tana workspace JSON export** into **Logseq OG** — offline, OCC-safe, idempotent, dry-run by default.

Matryca Plumber now streams Tana's flat `docs[]` graph dump via **`ijson`** (no full-DOM OOM), maps entities to `Tana/` pages, routes `#day` nodes through your vault's **`logseq/config.edn`** journal format, resolves `[[wikilinks]]` against the in-flight batch and existing catalog, and writes with fresh Logseq `id::` UUIDs plus **`tana-id::`** provenance for safe re-import.

**Why upgrade:** leaving Tana for Logseq OG no longer means hand-rebuilding supertags, fields, and calendar nodes. Operators get **`matryca import tana --file … [--apply]`** and MCP **`import_tana`** on the same mutation plane as `ingest_document` — page locks, OCC, secret scan, optional robot git commits.

Operators should upgrade with `uvx matryca-plumber` ≥ **1.11.0**. Point LLM hosts at **`llms.txt` v1.11.0**.

## Operator quick start

```bash
export LOGSEQ_GRAPH_PATH=/path/to/logseq/graph

# 1 — dry-run (review counters)
matryca import tana --file ~/Downloads/tana-workspace.json | jq '.write'

# 2 — apply on a test clone first
matryca import tana --file ~/Downloads/tana-workspace.json --apply
```

**Test vault first.** Duplicate your graph folder before `--apply`. See README § Clone your graph first.
