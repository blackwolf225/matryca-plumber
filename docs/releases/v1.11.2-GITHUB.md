# GitHub Release — v1.11.2 (copy-paste)

**Title:** v1.11.2 - Graph Layer Boundary Refactor

**Tag:** `v1.11.2`

## Summary

**Architecture release:** canonical graph primitives move into `src/graph/` with enforced layer boundaries (`tests/test_graph_layer_boundary.py`). The inverted graph→daemon post-write dependency is resolved via a graph-local `PageWrittenEvent` port ([#134](https://github.com/MarcoPorcellato/matryca-plumber/issues/134)).

**Memory safety:** bounded LRU pools for generational alias/BM25 caches and dual-embedding block vectors — ondemand streaming is now the default ([#136](https://github.com/MarcoPorcellato/matryca-plumber/issues/136), [#51](https://github.com/MarcoPorcellato/matryca-plumber/issues/51) partial).

**Concurrency:** page writes use `st_mtime_ns` nanosecond OCC snapshots ([#153](https://github.com/MarcoPorcellato/matryca-plumber/issues/153) partial); `graph_dispatch` resolve/write TOCTOU closed ([#133](https://github.com/MarcoPorcellato/matryca-plumber/issues/133)).

Operators should upgrade with `uvx matryca-plumber` ≥ **1.11.2**. Point LLM hosts at **`llms.txt` v1.11.2**.

## Added

- Graph layer ports and `tests/test_graph_layer_boundary.py`.
- Generational cache LRU (`MATRYCA_GENERATIONAL_CACHE_MAX_GRAPHS`).
- Dual-embedding ondemand/resident modes with page-scoped streaming merge.
- Read-only `read_daemon_checkpoint()` for Soft Gate / MCP bootstrap status.
- `SemanticRuntimeConfig` for injectable semantic indexer settings.
- Phase 2 progress baseline (`phase2_vault_baseline_total`) for stable Sovereign UI denominator.

## Changed

- Agent → graph moves with backward-compatible re-exports.
- `bootstrap_status` uses lightweight checkpoint reader.
- Generational alias build retry (3 attempts) for concurrent-write sig races.
- Journal page detection in graph layer ([#71](https://github.com/MarcoPorcellato/matryca-plumber/issues/71) partial).

## Fixed

- Lock backoff downgrade when file already processed ([#132](https://github.com/MarcoPorcellato/matryca-plumber/issues/132)).
- TUI daemon state load narrow catch + breadcrumb ([#130](https://github.com/MarcoPorcellato/matryca-plumber/issues/130)).
- Journal link registry merge logging ([#131](https://github.com/MarcoPorcellato/matryca-plumber/issues/131)).
- CI `httpx2` dev dependency silences Starlette deprecation warnings ([#118](https://github.com/MarcoPorcellato/matryca-plumber/issues/118), @blackwolf225).
