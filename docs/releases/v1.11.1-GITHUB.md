## Summary

**Parser alignment patch:** raises the minimum **`logseq-matryca-parser`** dependency to **1.4.0** so Plumber inherits the latest graph-integrity and robustness fixes from the Logos Protocol release line.

**Why upgrade:** parser **1.4.0** ships 31 bug-hunt fixes, canonical page iteration, case-insensitive tag/search, live watcher delete/move handling, and SYNAPSE embed safety — all on the same headless OCC mutation plane as v1.11.0 Tana import. No Plumber API changes required.

Operators should upgrade with `uvx matryca-plumber` ≥ **1.11.1**. Point LLM hosts at **`llms.txt` v1.11.1**.

## Highlights

| Area | Change |
|------|--------|
| Parser pin | `logseq-matryca-parser>=1.4.0,<2.0.0` (was `>=1.3.1`) |
| Inherited fixes | Canonical page iteration; case-insensitive tag/search; watcher delete/move; SYNAPSE embed safety |
| CI hygiene | Ruff format on Tana importer modules/tests |

## Changed

- **`logseq-matryca-parser`** — minimum dependency **1.4.0**; inherits 1.3.x public API + graph parity.
- **Ruff format** — `tags.py`, `write.py`, `test_tana_convert.py`, `test_tana_tags.py`.
