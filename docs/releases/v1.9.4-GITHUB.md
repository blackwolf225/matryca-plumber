# GitHub Release — v1.9.4 (copy-paste)

**Title:** v1.9.4 - The "Clean Journal" Update

**Tag:** `v1.9.4`

---

## One bullet per day

The daemon no longer appends a new `## 🤖 Matryca Activity` section after every duty cycle. Instead it **upserts a single cumulative line** in today's journal:

```markdown
- 🤖 Matryca Activity — indexed 12 page(s); checked 340 link(s); flagged 2 block(s); 47 duty cycle(s)
```

Daily totals increment in place — no more 50–200 journal lines from background polling.

## Smarter writes

- **Idle skip:** cycles with no indexing, fast-track, link checks, or flags do not touch the journal.
- **Daemon ledger:** `DaemonState.journey_day` in `.matryca_daemon_state.json` is the source of truth; resets at calendar day change.
- **Legacy cleanup:** old `## 🤖 Matryca Activity` blocks on today's file are removed on the first upsert after upgrade.

## Operator control

Disable entirely with `MATRYCA_JOURNEY_LOG_ENABLED=false` (documented in `.env.example`).
