# GitHub Release — v1.9.15 (copy-paste)

**Title:** v1.9.15 - Strict Mypy Compliance & Journal Phase-2 Bypass

**Tag:** `v1.9.15`

---

## Summary

**Quality & efficiency release:** closes [#60](https://github.com/MarcoPorcellato/matryca-plumber/issues/60) with zero `# type: ignore` cheats under `src/`, and extends journal-aware Phase 2 so daily notes skip semantic LLM indexing and dual embeddings entirely — structural Phase 1 only.

**Why upgrade:** journal-heavy vaults spend fewer LLM tokens on daily notes that v1.9.14 already excluded from Louvain clustering; contributors and CI get a strictly typed codebase with no suppressed mypy violations in production modules.

Operators should upgrade with `uvx matryca-plumber` ≥ **1.9.15**.

## Highlights

| Area | Change |
|------|--------|
| Type safety (#60) | All 11 `# type: ignore` suppressions removed from `src/`; strict mypy passes via narrowing, `cast()`, and a `_FilesystemObserver` Protocol |
| Journal Phase 2 | `journals/` pages bypass semantic indexing and dual embeddings; Phase 1 (AST cache, link registry, OCC `mtime` ledger) still runs |
| Token efficiency | Complements v1.9.14 clustering skip — daily notes no longer incur per-page embedding LLM work |

## Fixed

- **Mypy strictness (#60)** — No `# type: ignore` under `src/`; `isinstance()` guards, `Path()` coercion, and typed observer protocol replace suppressions.

## Changed

- **Journal Phase-2 skip** — Maintenance daemon short-circuits semantic indexing for `journals/` paths after Phase 1 structural work (`src/agent/maintenance_daemon.py`).
