# GitHub Release — v1.10.6 (copy-paste)

**Title:** v1.10.6 - Concurrency Integrity (Unified Flock + Hub Page OCC)

**Tag:** `v1.10.6`

---

## Summary

**Concurrency integrity patch:** unifies cross-process flock semantics for page RMW locks and JSON sidecar locks, and adds OCC-safe writes for daemon-generated hub pages (Master Index, Graph Insights).

**Why upgrade:** fixes nested catalog/registry deadlocks when harvest, daemon checkpoint, and link-registry passes interleave ([#40](https://github.com/MarcoPorcellato/matryca-plumber/issues/40)); prevents silent overwrite of human edits on hub pages during multi-second compiles ([#34](https://github.com/MarcoPorcellato/matryca-plumber/issues/34)). Builds on v1.10.5 parser 1.3.1 alignment.

Operators should upgrade with `uvx matryca-plumber` ≥ **1.10.6**. Point LLM hosts at **`llms.txt` v1.10.6**.

## Highlights

| Area | Change |

| Unified flock | `src/utils/platform_lock.py` — NB acquire + backoff + reentrancy shared by page + JSON locks |

## Fixed

- **JSON sidecar flock parity (#40)** — `cross_process_json_flock` shares `platform_lock` with page RMW locks.
- **Hub page OCC (#34)** — Master Index and Graph Insights compiles use OCC + graceful skip.
