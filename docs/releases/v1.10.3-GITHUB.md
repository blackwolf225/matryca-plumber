# GitHub Release — v1.10.3 (copy-paste)

**Title:** v1.10.3 - Infrastructure Hardening & Sovereign UI Resilience

**Tag:** `v1.10.3`

---

## Summary

**Operator-facing patch:** the Sovereign UI no longer freezes telemetry polling while saving settings or graph paths; LLM structured-output contracts are stricter for MCP outline mutations and plumber cognitive modules; flock sidecar files are created with tighter permissions.

**Why upgrade:** if the control room felt stuck during **Save** or settings drawer edits, this release offloads blocking `.env` I/O and runtime bootstrap to a worker thread. LLM hosts benefit from recursive strict JSON schemas and `extra="forbid"` on outline/plumber response models. Local inference servers (LM Studio/Ollama) keep using `max_tokens`; o-series/gpt-5 style models use `max_completion_tokens`.

Operators should upgrade with `uvx matryca-plumber` ≥ **1.10.3**. Point LLM hosts at **`llms.txt` v1.10.3**.

## Highlights

| Area | Change |
|------|--------|
| Sovereign UI | Non-blocking `POST /api/config` and graph-path saves; rotating Loguru file logs at UI startup |
| Model discovery | `GET /api/lm-models` uses `httpx` with redirect refusal (SSRF-aligned) |
| LLM contracts | Pydantic `extra="forbid"` on plumber/outline models; recursive strict JSON Schema via OpenAI SDK helper |
| Token caps | Adaptive `max_tokens` vs `max_completion_tokens` by model id |
| Security | Flock sidecars (`json_flock`, page RMW, daemon lock) created as `0o600` |
| CI | CodeQL suite narrowed to `security-extended` |

## Changed

- **Sovereign UI API** — config/graph-path saves run off the asyncio event loop; LM Studio discovery via `httpx`; `configure_loguru()` at control-room startup.
- **LLM structured output** — strict nested schemas; forbidden extra keys on structured models; adaptive completion token parameter naming.
- **CodeQL** — security-focused query suite only.

## Security

- **Lock sidecar permissions** — cross-process flock files created with `0o600` instead of `0o644`.
